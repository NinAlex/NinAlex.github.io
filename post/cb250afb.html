基于Hexo和github建设个人博客

Created2023-02-01|Updated2023-02-01
|Post View:


Hexo是一个快速、间接且高效的博客框架，通过使用Hexo我们能够快速的建立一个属于我们自己的个人Blog！

开始前的准备

创建一个github项目

利用github项目作为仓库来存储你上传到公共网络的博客静态页面
注意，项目仓库的名字必须是你github的注册名，即项目名称设定为 你的Github名.github.io 这个格式
同时你注册的密码和github绑定的邮箱在SSH也有用，务必记住
（在建立项目仓库时记得选择 Add a README file, 这是给你项目增加一份说明文件）
然后点击Creat，你就建立了一个github项目了（这可能对大部分人来说是人生的第一个github项目）

安装配置Node.js

Node.js能够使JavaScript成为与PHP、Python、Perl、Ruby等服务端语言平起平坐的脚本语言。
下载链接：https://nodejs.org/ja/

可能会遇到的问题（待更新）

安装并配置Git

Git是一个分布式版本控制系统，用来进行项目版本管理
下载地址：https://git-scm.com/downloads
安装过程中设定好安装目标文件夹之后无脑next就好~

使用Git安装Hexo

打开 Git bash 输入命令 npm install hexo-cli -g

正式开始

生成SSH并绑定Github账户

打开 git Bash 并输入生成SSH命令

在双引号填入你注册Github账号时的邮箱地址，加下来运行该命令。

接下来会运行过程中需要你敲击三次空格，其中有一次会对你进行询问，只需填写y，然后回车继续运行。

运行结束以后，我们去找生成的SSH
位置是C:\Users(有的系统文件夹为中文“用户”)\你的Windows用户名文件夹.ssh
在该路径下用文本文档方式打开 id_rsa.pub 文件

打开之后文本文档会有一长串SSH密钥，全选并复制SSH密钥

打开github，登录账号，进入setting

在setting中选择 SSH and GPG keys 选项

选择New SSH key

Title的名称随意，Key type默认即可无需调整，将你复制的SSH密钥粘贴到Key中，随后不需要做任何改动，只需要直接点击 Add SSH key即可。

上传的问题

此处是问题的频发区，主要存在的问题有：

1.网络连接问题

重新输入hexo d命令上传，或者检查网络，最近github被墙了，可能会影响国内使用

2.hexo d 指令无法执行

尝试使用以上全局指令，同时这也是当本地预览和上传页面出现不同时的解决方法。

另外可以尝试

先输入：
git config –global user.email “你的邮箱”
git config –global user.name “你的gihub名字”
然后
hexo clean
hexo g
hexo d

如果hexo d不动，就用ctrl+c终止，再重新使用hexo d 开始上传
